# Evidex Parachain Setup and Usage Guide

## 🚀 Quick Setup

### 1. Prerequisites
- Docker Desktop running
- Node.js 18+
- pnpm package manager
- Git

### 2. Initial Setup
```bash
# Clone and navigate to project
cd /Users/adityashrivastava/Desktop/BLOCKCHAIN

# Install dependencies
pnpm install

# Setup parachain environment
pnpm parachain:setup
```

### 3. Start Infrastructure
```bash
# Start database, cache, and message queue
docker compose up -d postgres redis rabbitmq

# Verify services are running
docker compose ps
```

### 4. Start Evidex Application
```bash
# Start the development server
pnpm dev
```

## 🏗️ Parachain Network Options

### Option 1: Local Development (Recommended)
```bash
# Start minimal relay chain
docker-compose -f docker-compose.parachain.minimal.yml up -d

# Check status
docker-compose -f docker-compose.parachain.minimal.yml ps
```

### Option 2: Full Parachain Network
```bash
# Start complete network (requires building parachain image)
docker-compose -f docker-compose.parachain.yml up -d

# View logs
docker-compose -f docker-compose.parachain.yml logs -f
```

- **API Gateway**: http://localhost:3000/api

- **PostgreSQL**: localhost:5432
- **Redis**: localhost:6379
- **RabbitMQ Management**: http://localhost:15672

## 🔗 Share Admin/User Generated Blocks

Use these URLs to view and share only the blocks generated by each role:

- All generated blocks: `http://localhost:3000/explorer`
- Admin generated blocks: `http://localhost:3000/explorer?role=ADMIN`
- User generated blocks: `http://localhost:3000/explorer?role=USER`

Export filtered results:

- Admin CSV: `http://localhost:3000/api/explorer?role=ADMIN&format=csv`
- User CSV: `http://localhost:3000/api/explorer?role=USER&format=csv`

Live chain details for one transaction:

- `http://localhost:3000/api/chain/tx/<txHash>?chain=polkadot`

## 📝 Using the Evidence Pallet

### 1. Register Evidence
```javascript
// Using the Evidex app
const evidence = {
  file: yourFile,
  metadata: {
    title: "Evidence Title",
    description: "Evidence Description"
  }
};

// This will:
// 1. Generate SHA256 hash
// 2. Upload to IPFS via Pinata
// 3. Register on parachain with fee
// 4. Store metadata in PostgreSQL
```

### 2. Verify Evidence
```javascript
// Verify by file upload or hash
const verification = await verifyEvidence({
  hash: "0x1234567890abcdef...",
  // or file: yourFile
});

// Returns verification result with blockchain proof
```

### 3. Direct Pallet Interaction
```bash
# Using polkadot.js.org/apps
# Connect to: ws://localhost:9933

# Call submit_evidence (or register_evidence alias)
# Parameters:
# - file_hash: Vec<u8> (SHA256 hex bytes, 64 chars)
# - ipfs_cid: Vec<u8> (IPFS content identifier)

# Call verify_evidence
# Parameters:
# - evidence_id: H256 (Blake2 hash of file_hash bytes)

# Optional lifecycle calls:
# - challenge_evidence(evidence_id, reason)
# - resolve_challenge(evidence_id, accept)
```

## 🔍 Evidence Pallet Functions

### `submit_evidence(file_hash, ipfs_cid)` / `register_evidence(file_hash, ipfs_cid)`
- **Purpose**: Register new evidence on-chain
- **Fee**: 0.001 EVID tokens
- **Validation**: Unique hash, valid 64-char SHA256 hex, valid IPFS CID
- **Storage**: Evidence registry with owner, timestamp, CID

### `verify_evidence(evidence_id)` - Admin Only
- **Purpose**: Mark evidence as verified
- **Authorization**: Requires admin key
- **Use Case**: Approve pending/challenged evidence

### `challenge_evidence(evidence_id, reason)`
- **Purpose**: Raise a challenge against submitted/verified evidence
- **Authorization**: Owner or admin
- **State Change**: Status becomes `Challenged`

### `resolve_challenge(evidence_id, accept)` - Admin Only
- **Purpose**: Resolve challenge to `Verified` (`accept=true`) or `Rejected` (`accept=false`)
- **Authorization**: Requires admin key
- **State Change**: Removes active challenge record

## 🚨 Troubleshooting

### Common Issues

1. **Port Conflicts**
```bash
# Check what's using ports
lsof -i :3000 -i :5432 -i :6379 -i :9933

# Kill conflicting processes
kill -9 <PID>
```

2. **Docker Issues**
```bash
# Reset Docker
docker system prune -a
docker volume prune
```

3. **Database Connection**
```bash
# Reset database
docker compose down postgres
docker volume rm blockchain_postgres_data
docker compose up -d postgres
pnpm db:migrate
pnpm db:seed
```

4. **Parachain Not Starting**
```bash
# Check logs
docker-compose -f docker-compose.parachain.minimal.yml logs

# Restart services
docker-compose -f docker-compose.parachain.minimal.yml restart
```

### Debug Commands

```bash
# Check parachain health
curl http://localhost:9933/health

# Test API endpoints
curl http://localhost:3000/api/auth/challenge

# Check database connection
pnpm db:generate
```
